feat(feed-scorer): implement native TypeScript feed scoring engine with request-time personalization, and restore match-generator + notification-engine service exports

This commit completes the transition from Python-worker-backed services to fully
native TypeScript implementations for feed curation, match generation, and
notifications. No Docker, cron jobs, or external services required.

## What was built

### 1. Feed Scorer Engine (lib/services/feed-scorer.ts) — NEW
- seededThompsonSample() — Deterministic Thompson Sampling using Beta distribution
  approximation (LCG PRNG + Box-Muller transform, mirrors removed Python FeedScorer)
- calculateHybridScore() — Weighted scoring: semantic (35%) + engagement via
  Thompson Sampling (30%) + recency (20%), with connection (1.5x), shared
  interests (1.2x), and intent match (1.1x) boosts, capped at 1.0
- calculateRecencyScore() — Exponential decay with 24h half-life
- scorePostForUser() / scoreFeedForUser() — Batch scoring with factor breakdown
- persistFeedScores() / cleanupExpiredFeedScores() — Optional Supabase persistence
- 23 unit tests all passing

### 2. Request-Time Feed Personalization (lib/services/posts.ts) — REWRITTEN
fetchPersonalizedFeed() was previously querying the feed_scores table which was
always empty (no pipeline to populate it), so it silently fell back to chronological
ordering. Now it computes scores at request-time from live database data:

1. Checks viewer's profile_embeddings — falls back to chronological if none
2. Batches 5 Supabase queries: viewer embedding, viewer interests/looking_for,
   recent posts (50), author embeddings, author interests, connection status
3. For each post computes: cosine similarity (viewer vs author embeddings),
   connection check (connections table), shared interests overlap, intent match
   (looking_for vs post.intent), recency decay
4. Feeds all factors into calculateHybridScore(), returns top N sorted by score

### 3. API Route (app/api/feed/score/route.ts) — NEW
POST endpoint with CSRF protection and rate limiting. Accepts post scoring
inputs, returns scored + ranked results, optionally persists to feed_scores table.

### 4. React Query Hook (hooks/use-feed.ts) — NEW
fetchFeed() and useScoreFeed() mutation for client-side scoring.

### 5. Service Exports (lib/services/index.ts) — MODIFIED

all three native TS services are importable from the barrel file.

### 6. Feed Component (components/features/dashboard/feed.tsx) — MODIFIED
